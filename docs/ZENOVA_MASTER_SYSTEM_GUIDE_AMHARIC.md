# የZENOVA ማስተር ሲስተም መመሪያ መጽሐፍ

## ምዕራፍ ፩ — ZENOVA ምንድን ነው?

### 1.1 መግቢያ

ZENOVA የትምህርት ተቋማትን ሙሉ በሙሉ በዲጂታል መንገድ ለማስተዳደር የተሰራ የተቀናጀ የትምህርት ቴክኖሎጂ (EdTech) መድረክ ነው። ይህ ሲስተም ከትምህርት ቤት አያያዝ ጀምሮ እስከ ፋይናንስ፣ ቤተ-መጻሕፍት፣ ካፍቴሪያ፣ መደብር፣ የተማሪ ክትትል፣ የደህንነት ስርዓት እና የፍቃድ አሰጣጥ ድረስ ያሉትን ሁሉንም የትምህርት ተቋም እንቅስቃሴዎች በአንድ ጣሪያ ስር ያስተዳድራል።

### 1.2 የሲስተሙ ዋና ዋና ባህሪያት

ZENOVA ከሌሎች የትምህርት ማኔጅመንት ሲስተሞች የሚለየው በሚከተሉት ባህሪያቱ ነው፦

- **የ13+ ሚናዎች ድጋፍ** — ከሱፐር አድሚን እስከ ተማሪ ድረስ ለእያንዳንዱ የትምህርት ቤት ባለድርሻ የተለየ ዳሽቦርድ እና ፍቃድ ይሰጣል።
- **የ3-ደረጃ አርክቴክቸር** — ደመና (Cloud VPS)፣ የትምህርት ቤት አገልጋይ (School Ubuntu Server)፣ እና የተጠቃሚ ፊት ለፊት (Frontend) በሚል መዋቅር የተከፋፈለ ነው።
- **NFC/QR ክትትል** — የተማሪዎችን መገኘት፣ የካፍቴሪያ ክፍያ እና የደህንነት ፍተሻ በNFC ካርዶች እና QR ኮዶች ማከናወን ይቻላል።
- **የበርካታ ክፍያ አማራጮች** — ቻፓ፣ ቴሌብር እና ሲቢኢ ብርን ጨምሮ የተለያዩ የክፍያ መንገዶችን ይደግፋል።
- **የፍቃድ አሰጣጥ ሲስተም** — እያንዳንዱ ትምህርት ቤት በፍቃድ (license) መሰረት ሲስተሙን መጠቀም ይችላል።
- **ከ229 በላይ ገፆች** — ለተለያዩ ሚናዎች እና ተግባራት የተዘጋጁ ገፆች።

### 1.3 የሲስተሙ አላማ

ZENOVA የተፈጠረበት ዋና አላማ የትምህርት ተቋማትን የአስተዳደር ስራ በዲጂታል መንገድ በማዘመን ውጤታማነትን ማሳደግ ነው። በተለይም በኢትዮጵያ እና በሌሎች ታዳጊ ሀገራት የትምህርት ተቋማት የሚገጥሟቸውን እንደ፡

- የተማሪ መረጃ አያያዝ ችግር
- የክፍያ አሰባሰብ እና ክትትል ተግዳሮት

- የውጤት አስተዳደር

ያሉ ችግሮችን ለመፍታት ያለመ ነው።

## ምዕራፍ ፪ — የሲስተሙ አርክቴክቸር (System Architecture)

### 2.1 የ3-ደረጃ አርክቴክቸር መግለጫ

ZENOVA በሶስት ዋና ዋና ደረጃዎች የተከፋፈለ አርክቴክቸር ይከተላል። ይህ የሶስት-ደረጃ አሰራር ሲስተሙን ደህንነቱ የተጠበቀ፣ ተከሳሽ እና በቀላሉ ሊስፋፋ የሚችል ያደርገዋል።

#### ደረጃ ፩፦ ደመና (Cloud VPS)

ይህ የማዕከላዊ የውሂብ ማዕከል ሲሆን በበይነ መረብ ላይ የሚገኝ ምናባዊ የግል አገልጋይ (Virtual Private Server) ነው። ተግባሮቹ፦

- የሁሉም ትምህርት ቤቶች ማዕከላዊ ዳታቤዝ
- የፍቃድ አሰጣጥ እና ማረጋገጫ (License Server)
- የደመና ማመሳሰል (Cloud Sync)
- የሱፐር አድሚን ዳሽቦርድ
- የChapa፣ Telebirr እና CBE Birr የክፍያ መጋራቢያ (Payment Gateway)
- የወላጆች እና ተማሪዎች ርቀት መዳረሻ

#### ደረጃ ፪፦ የትምህርት ቤት አገልጋይ (School Server)

እያንዳንዱ ትምህርት ቤት በራሱ ተቋም ውስጥ የሚገኝ የኡቡንቱ (Ubuntu) አገልጋይ አለው። ተግባሮቹ፦

- የትምህርት ቤቱ የውስጥ ዳታቤዝ
- የተማሪ መረጃ አያያዝ (አካዳሚክ፣ ፋይናንስ፣ ቤተ-መጻሕፍት)
- የNFC እና QR ኮድ ማስኬጃ
- ከደመና ጋር ወቅታዊ ማመሳሰል
- የአካባቢ ደህንነት እና ምትኬ (Local Backup)

#### ደረጃ ፫፦ የተጠቃሚ ፊት ለፊት (Frontend)

ይህ ተጠቃሚዎች በብራውዘር ወይም በሞባይል አፕሊኬሽን የሚጠቀሙበት የሲስተሙ ገጽታ ነው። የተለያዩ ተጠቃሚዎች በሚናቸው መሰረት የተለየ ዳሽቦርድ እና ገፆች ያያሉ።

### 2.2 የውሂብ ፍሰት (Data Flow)

የውሂብ ፍሰቱ በሶስቱ ደረጃዎች መካከል እንደሚከተለው ነው፦

1. **ተጠቃሚ ወደ ፊት-ለፊት** — ተጠቃሚው በብራውዘር ወደ ሲስተሙ ገጽ ይገባል
2. **ፊት-ለፊት ወደ አገልጋይ** — ጥያቄው ወደ ትምህርት ቤቱ አገልጋይ ወይም ወደ ደመና ይላካል
3. **አገልጋይ ወደ ዳታቤዝ** — አገልጋዩ ዳታቤዙን አነጋግሮ ውጤቱን ይመልሳል
4. **አገልጋይ ወደ ፊት-ለፊት** — ውጤቱ ወደ ተጠቃሚው ብራውዘር ይመለሳል
5. **ደመና ማመሳሰል** — የትምህርት ቤቱ አገልጋይ በየጊዜው ከደመና ጋር ይሰማማል

### 2.3 የቴክኖሎጂ ቁልል (Tech Stack)

ZENOVA ዘመናዊ የሆኑ ቴክኖሎጂዎችን በመጠቀም የተሰራ ነው፦

**የፊት-ለፊት ክፍል (Frontend)፦**
- Next.js 14 (React Framework) — የተጠቃሚ በይነገጽ መሰረት
- TypeScript — ደህንነቱ የተጠበቀ የኮድ አጻጻፍ
- Tailwind CSS — ፈጣን እና ምላሽ ሰጪ ዲዛይን
- React Query — የውሂብ ማምጣት እና ማስተዳደር
- ESLint — የኮድ ጥራት መመሪያ

**የኋላ-ተናጋሪ ክፍል (Backend)፦**
- Node.js — የአገልጋይ ፕሮግራሚንግ
- PostgreSQL — የውሂብ ጎታ
- REST API — የመገናኛ ዘዴ

**ሌሎች ቴክኖሎጂዎች፦**
- NFC Technology — የተማሪ መለያ ካርዶች
- QR Code — ፈጣን የተማሪ መለያ
- Ubuntu Server — የትምህርት ቤት አገልጋይ
- Docker — የአገልጋይ ማስተናገጃ

### 2.4 የደህንነት ንብርብሮች

ሲስተሙ በበርካታ የደህንነት ንብርብሮች የተከለለ ነው፦

1. **JWT ማረጋገጫ** — እያንዳንዱ ተጠቃሚ በሚገባበት ጊዜ የሚሰጠው የደህንነት ቶከን
2. **RBAC (Role-Based Access Control)** — በሚና ላይ የተመሰረተ የመዳረሻ ቁጥጥር
3. **የመካከለኛ ሶፍትዌር ጥበቃ (Middleware)** — ገፆችን ከማሳየቱ በፊት ፍቃድ ማረጋገጥ
4. **የውሂብ ምስጠራ** — ሚስጥራዊ ውሂብ በምስጠራ መቀመጥ
5. **የአገልጋይ ደህንነት** — አገልጋዮች በፋየርዎል እና በሌሎች መከላከያዎች የተከለሉ ናቸው

### 3.1 አጠቃላይ እይታ

አዲስ ትምህርት ቤት ZENOVA ሲስተም ውስጥ ለመመዝገብ የሚከተሉትን ደረጃዎች ያልፋል። ይህ ሂደት የሚከታተለው ሱፐር አድሚን ሲሆን እያንዳንዱ ትምህርት ቤት በሲስተሙ ውስጥ የራሱ የሆነ የተለየ ቦታ እንዲኖረው ያደርጋል።

### 3.2 የምዝገባ ሂደት

**ደረጃ 1፦ የትምህርት ቤት መረጃ ምዝገባ**
ሱፐር አድሚኑ ስለ ትምህርት ቤቱ የሚከተሉትን መረጃዎች ያስገባል፦
- ሙሉ ስም እና አህጽሮት ስም
- አድራሻ (ክልል፣ ዞን፣ ወረዳ፣ ቀበሌ)
- የስልክ ቁጥር እና ኢሜይል
- የትምህርት ቤቱ አይነት (የግል፣ የመንግስት፣ ወዘተ)
- የትምህርት ደረጃ (ከመዋዕለ ሕጻናት እስከ ሁለተኛ ደረጃ)

**ደረጃ 2፦ የፍቃድ አሰጣጥ**
- ለትምህርት ቤቱ ልዩ የሆነ የፍቃድ ቁልፍ (License Key) ይፈጠራል
- የፍቃዱ ዓይነት (Basic፣ Standard፣ Premium) ይወሰናል
- የፍቃዱ የሚቆይበት ጊዜ ይቀመጣል
- የሚፈቀደው የተማሪ ብዛት ይወሰናል

**ደረጃ 3፦ የአገልጋይ ውቅር (Server Setup)**
- የትምህርት ቤቱ የአካባቢ አገልጋይ ይዘጋጃል
- የዳታቤዝ ውቅር ይፈጠራል
- ከደመና ጋር የሚገናኝ የማመሳሰል ዘዴ ይቋቋማል

**ደረጃ 4፦ የመጀመሪያ አድሚን መፍጠር**
- የትምህርት ቤቱ የመጀመሪያ አድሚን አካውንት ይፈጠራል
- የሚገባበት መረጃ (username እና ጊዜያዊ የይለፍ ቃል) ይሰጣል
- አድሚኑ በመጀመሪያ ግባታ የይለፍ ቃሉን እንዲቀይር ይጠበቅበታል

### 3.3 የትምህርት ቤቶች ዝርዝር እና አስተዳደር

ሱፐር አድሚኑ የሚከተሉትን የማየት እና የማስተዳደር ችሎታ አለው፦

- የሁሉም ትምህርት ቤቶች ዝርዝር ከአሁኑ ሁኔታቸው ጋር
- እያንዳንዱ ትምህርት ቤት ተማሪ መመዝገብ መቻሉን ማረጋገጥ
- የፍቃድ ማራዘሚያ እና ማደስ
- ትምህርት ቤት መዝጋት ወይም እንደገና ማንቃት
- የትምህርት ቤት መረጃ ማስተካከል

### 3.4 የአካዳሚክ ዘመን አስተዳደር

እያንዳንዱ ትምህርት ቤት የራሱን የአካዳሚክ ዘመን ያስተዳድራል፦

- **የአካዳሚክ ዘመን መፍጠር** — ለምሳሌ 2017 ዓ.ም. እስከ 2018 ዓ.ም.
- **የትምህርት ዘመን መክፈት/መዝጋት** — የክፍል ምዝገባን መቆጣጠር
- **የፈተና ዘመን** — የፈተና ፕሮግራም ማዘጋጀት
- **የውጤት አቀራረብ** — ውጤቶችን ለወላጆች የሚለቀቅበት ጊዜ

### 3.5 የክፍል እና ክፍልፋይ አስተዳደር

ትምህርት ቤቱ የራሱን መዋቅር እንደሚከተለው ያቋቁማል፦

- **ክፍሎች/ደረጃዎች** — ከቅድመ-መደበኛ እስከ 12ኛ ክፍል
- **ክፍልፋዮች (Sections)** — እንደ ክፍል A, B, C ወዘተ
- **የክፍል አስተማሪ መመደብ** — እያንዳንዱ ክፍልፋይ የራሱ የክፍል አስተማሪ (Homeroom Teacher) ይኖረዋል
- **የመማሪያ ክፍሎች** — የአካላዊ ክፍሎች አስተዳደር

## ምዕራፍ ፬ — የሱፐር አድሚን ዳሽቦርድ እና አስተዳደር

### 4.1 የሱፐር አድሚን ሚና ምንድን ነው?

ሱፐር አድሚን የZENOVA ሲስተም ከፍተኛው የአስተዳደር ሚና ነው። ይህ ሰው መላውን ሲስተም — ሁሉንም ትምህርት ቤቶች፣ ተጠቃሚዎች፣ ፍቃዶች እና የሲስተም ውቅሮችን — የማስተዳደር ሙሉ ስልጣን አለው።

### 4.2 ዋና ዋና ተግባራት

**የትምህርት ቤቶች አስተዳደር፦**
- አዲስ ትምህርት ቤት መመዝገብ
- የትምህርት ቤት መረጃ ማስተካከል
- ትምህርት ቤት ማንቃት/ማጥፋት
- የትምህርት ቤቶች ዝርዝር እና ሁኔታ መከታተል

**የፍቃድ አስተዳደር፦**
- ለአዳዲስ ትምህርት ቤቶች ፍቃድ መስጠት
- የነባር ፍቃዶች ማራዘሚያ
- ፍቃድ ማቋረጥ
- የፍቃድ አጠቃቀም ሪፖርት

**የሲስተም ክትትል፦**
- የአገልጋይ ሁኔታ መከታተል
- የደመና ማመሳሰል ሁኔታ
- የስህተት ሪፖርቶች
- ስታቲስቲካል ሪፖርቶች

### 4.3 የሱፐር አድሚን ዳሽቦርድ

የሱፐር አድሚን ዳሽቦርድ የሚከተሉትን ያሳያል፦

- **አጠቃላይ ስታቲስቲክስ** — ጠቅላላ ትምህርት ቤቶች፣ ተማሪዎች፣ አስተማሪዎች
- **ንቁ ፍቃዶች** — በአሁኑ ጊዜ የሚሰሩ ፍቃዶች
- **የወር አዲስ ምዝገባዎች** — በወር አቆጣጠር አዲስ የተመዘገቡ ትምህርት ቤቶች
- **የሲስተም ሁኔታ** — የአገልጋዮች እና የዳታቤዝ ሁኔታ
- **የቅርብ ጊዜ እንቅስቃሴዎች** — በሲስተሙ ላይ የተከናወኑ የቅርብ ጊዜ ለውጦች

### 4.4 የሪፖርት አያያዝ

ሱፐር አድሚኑ የሚከተሉትን ሪፖርቶች ማግኘት ይችላል፦

- የፍቃድ ጊዜ ማብቂያ ሪፖርት
- የደመና ማመሳሰል ሪፖርት
- የሲስተም ስህተት ሪፖርት

## ምዕራፍ ፮ — ዳይሬክተር (Director)

### 6.1 ሚና እና ሃላፊነት

ዳይሬክተር የትምህርት ቤቱ ዋና የአካዳሚክ እና የአስተዳደር ተቆጣጣሪ ነው። ይህ ሚና በትምህርት ቤት ባለቤት እና በሌሎች ሰራተኞች መካከል የድልድይ ሚና ይጫወታል።

### 6.2 ዋና ዋና ተግባራት

**የአካዳሚክ ተቆጣጣሪ፦**
- የትምህርት ፕሮግራም እና ሥርዓተ ትምህርት መከታተል
- የፈተና እና የውጤት አሰጣጥ ቁጥጥር
- የመምህራን አፈጻጸም ግምገማ
- የትምህርት ጥራት ማረጋገጥ

**የአስተዳደር ተቆጣጣሪ፦**
- የአድሚን እና ሬጅስትራር ስራ ክትትል
- የሰራተኞች ፈረቃ እና የእረፍት ጊዜ አስተዳደር
- የትምህርት ቤት ቀን መቁጠሪያ አያያዝ

**የሪፖርት አያያዝ፦**
- የተማሪ መገኘት ሪፖርት
- የውጤት ሪፖርት ለባለቤቱ
- የትምህርት ቤት አፈጻጸም ሪፖርት

### 6.3 የዳይሬክተር ዳሽቦርድ

ዳይሬክተሩ የሚከተሉትን ያሳያል፦

- አጠቃላይ የተማሪ እና የመምህራን ብዛት
- የቅርብ ጊዜ የመገኘት መቶኛ
- የፈተና መርሐ ግብር
- የክፍል አፈጻጸም ማነጻጸሪያ
- የሰራተኞች እንቅስቃሴ ምዝግብ

### 6.4 የአካዳሚክ ዘመን አስተዳደር

ዳይሬክተሩ የአካዳሚክ ዘመኑን እንደሚከተለው ያስተዳድራል፦

- የትምህርት ዘመን መክፈት እና መዝጋት
- የፈተና ፕሮግራም ማዘጋጀት
- የውጤት ልቀቅ መርሐ ግብር
- የበዓላት እና የእረፍት ጊዜ አስተዳደር

## ምዕራፍ ፯ — አድሚን (Admin)

### 7.1 ሚና እና ሃላፊነት

አድሚን የትምህርት ቤቱ ዕለታዊ የአስተዳደር ስራዎችን የሚያከናውን ሰው ነው። ይህ ሚና የሲስተሙን የተለያዩ ክፍሎች በማስተናገድ የትምህርት ቤቱን ለስላሳ አሰራር ያረጋግጣል።

### 7.2 ዋና ዋና ተግባራት

**የተማሪ አስተዳደር፦**
- አዲስ ተማሪ መመዝገብ
- የተማሪ መረጃ ማስተካከል
- ተማሪ ወደ ሌላ ክፍል ማዛወር
- ተማሪ ማቋረጥ
- የተማሪ ፎቶ እና ሰነዶች አያያዝ

**የሰራተኛ አስተዳደር፦**
- አዲስ ሰራተኛ መመዝገብ
- የሰራተኛ መረጃ ማስተካከል
- የሰራተኛ ፈረቃ እና የስራ ጊዜ አስተዳደር

**የክፍል አስተዳደር፦**
- አዲስ ክፍል መፍጠር
- ተማሪዎችን ወደ ክፍሎች መመደብ
- የክፍል አስተማሪ መመደብ

### 7.3 የአድሚን ዳሽቦርድ

የአድሚን ዳሽቦርድ የሚከተሉትን ያሳያል፦

- የዛሬው የተማሪ መገኘት
- በምዝገባ ላይ ያሉ አዳዲስ ተማሪዎች
- ለማስተካከል የሚፈልጉ መረጃዎች
- የቅርብ ጊዜ እንቅስቃሴዎች
- ፈጣን አዶዎች ወደ ተደጋጋሚ ተግባራት

### 7.4 የሰራተኛ ፈረቃ እና የእረፍት አስተዳደር

አድሚኑ የሰራተኞችን ፈረቃ እና የእረፍት ጊዜ እንደሚከተለው ያስተዳድራል፦

- የስራ ፈረቃ መወሰን (ጠዋት፣ ከሰዓት፣ ሙሉ ቀን)
- የእረፍት ጊዜ ጥያቄ መቀበል እና ማስተናገድ
- የእረፍት ጊዜ ሚዛን ክትትል
- የሰራተኞች መገኘት ሪፖርት

ሬጅስትራር የተማሪ ምዝገባ፣ የውጤት አያያዝ እና የአካዳሚክ መዛግብትን የማስተዳደር ሃላፊነት አለው። ይህ ሚና ለትምህርት ቤቱ የአካዳሚክ መረጃ ትክክለኛነት እና ደህንነት ከፍተኛ ሃላፊነት አለው።

### 8.2 ዋና ዋና ተግባራት

**የተማሪ ምዝገባ፦**
- አዲስ ተማሪ መመዝገብ (ግላዊ መረጃ፣ የወላጅ መረጃ፣ አድራሻ)
- የተማሪ ሰነዶች ማያያዝ (የልደት የምስክር ወረቀት፣ ፎቶ፣ የቀድሞ ውጤት)
- ተማሪን ወደ ተገቢው ክፍል መመደብ
- የተማሪ መለያ ቁጥር (Student ID) መስጠት
- የNFC ካርድ መስጠት

**የውጤት አስተዳደር፦**
- የፈተና ውጤቶችን ማስገባት እና ማረጋገጥ
- የውጤት ደብተር ማዘጋጀት
- የውጤት ሪፖርት ካርድ ማ dručenje
- የትምህርት ቅጂ (Transcript) ማዘጋጀት

**የአካዳሚክ መዛግብት፦**
- የተማሪዎችን የትምህርት ታሪክ መዝገብ
- የተማሪ እንቅስቃሴ ምዝግብ (ማዛወር፣ ማቋረጥ፣ መመለስ)
- የተመራቂዎች ምዝገባ
- ስታቲስቲካል ዘገባዎች

### 8.3 የሬጅስትራር ዳሽቦርድ

የሬጅስትራር ዳሽቦርድ የሚከተሉትን ያሳያል፦

- ዛሬ የተመዘገቡ አዳዲስ ተማሪዎች
- ወደ ማረጋገጥ የሚጠብቁ ውጤቶች
- ሙሉ የተማሪ ብዛት በክፍል
- ያልተሟላ መረጃ ያላቸው ተማሪዎች
- የቅርብ ጊዜ የተማሪ እንቅስቃሴ

## ምዕራፍ ፱ — ፋይናንስ (Finance) — ዝርዝር መመሪያ

### 9.1 የፋይናንስ ሞጁል አጠቃላይ እይታ

የፋይናንስ ሞጁል የZENOVA ሲስተም ዋነኛ እና በጣም ዝርዝር ከሆኑት ክፍሎች አንዱ ነው። ይህ ሞጁል የትምህርት ቤቱን የገንዘብ ፍሰት ከመጀመሪያ እስከ መጨረሻ የሚቆጣጠር ሲሆን ከተማሪ ክፍያ አሰባሰብ ጀምሮ እስከ ደመወዝ አከፋፈል፣ የበጀት እቅድ እና የሂሳብ ሪፖርት ድረስ ያሉትን ሁሉንም የፋይናንስ ተግባራት ያከናውናል።

### 9.2 የክፍያ መዋቅር (Fee Structure)

የፋይናንስ ሞጁል የመጀመሪያው እና በጣም አስፈላጊው ክፍል የክፍያ መዋቅር ነው። ይህ እንዴት እንደሚሰራ እነሆ፦

**የክፍያ ዓይነቶች መፍጠር፦**
ትምህርት ቤቱ የሚሰበስባቸውን የተለያዩ የክፍያ ዓይነቶች መግለፅ ይችላል፦

- **የትምህርት ክፍያ (Tuition Fee)** — ዋናው ወርሃዊ/ዓመታዊ ክፍያ
- **የምዝገባ ክፍያ (Registration Fee)** — በየዓመቱ አንድ ጊዜ የሚከፈል
- **የላብራቶሪ ክፍያ (Lab Fee)** — ለሳይንስ ላብራቶሪ
- **የቤተ-መጻሕፍት ክፍያ (Library Fee)** — ለቤተ-መጻሕፍት አገልግሎት
- **የትራንስፖርት ክፍያ (Transport Fee)** — ለትምህርት ቤት አውቶቡስ
- **የምግብ ክፍያ (Meal Fee)** — ለካፍቴሪያ
- **የዩኒፎርም ክፍያ (Uniform Fee)** — ለዩኒፎርም
- **የፈተና ክፍያ (Exam Fee)** — ለፈተና ወረቀት እና አዘጋጅ
- **የልማት ክፍያ (Development Fee)** — ለትምህርት ቤት መሰረተ ልማት
- **ሌሎች (Other Fees)** — ተጨማሪ ክፍያዎች

**የክፍያ መዋቅር ውቅር፦**
እያንዳንዱ የክፍያ ዓይነት ለተለያዩ ክፍሎች እንዴት እንደሚተገበር ማዘጋጀት ይቻላል፦

- በክፍል ደረጃ (ለምሳሌ ከቅድመ-መደበኛ እስከ 12ኛ ክፍል)
- በተማሪ አይነት (አዲስ/ነባር ተማሪ)
- በክፍያ ድግግሞሽ (ወርሃዊ/ሩብ ዓመታዊ/ግማሽ ዓመታዊ/ዓመታዊ)
- በቅናሽ ዓይነት (በቡድን/ግለሰብ/ተደጋጋሚ)

### 9.3 የክፍያ አሰባሰብ (Payment Collection)

**የክፍያ አሰባሰብ ሂደት፦**

1. **የክፍያ መጠን መወሰን** — ሲስተሙ የተማሪውን ክፍል እና የተመረጡ አገልግሎቶች መሰረት በማድረግ የሚከፈለውን መጠን ያሰላል
2. **የክፍያ መጠየቂያ (Invoice)** — ሲስተሙ አውቶማቲክ የክፍያ መጠየቂያ ያመነጫል
3. **ክፍያ መቀበል** — ክፍያው በጥሬ ገንዘብ፣ በባንክ ዝውውር፣ በቻፓ፣ በቴሌብር ወይም በሲቢኢ ብር ሊከፈል ይችላል
4. **የክፍያ ማረጋገጫ** — ክፍያው ከገባ በኋላ የክፍያ መረጋገጫ (Receipt) ይታተማል
5. **የክፍያ ምዝገባ** — ክፍያው በሲስተሙ ውስጥ ተመዝግቦ የተማሪው የክፍያ ሁኔታ ይታደሳል

### 9.4 የክፍያ መዝገብ (Payment Ledger)

እያንዳንዱ ተማሪ የራሱ የክፍያ መዝገብ አለው ይህም የሚከተሉትን ያካትታል፦

- **የሚከፈል አጠቃላይ** — መከፈል ያለበት ጠቅላላ መጠን
- **የተከፈለ** — እስካሁን የተከፈለው መጠን
- **ቀሪ ሂሳብ** — መከፈል ያለበት ቀሪ መጠን
- **የክፍያ ታሪክ** — ቀደም ሲል የተከፈሉ ክፍያዎች ዝርዝር
- **የክፍያ መርሐ ግብር** — ቀጣይ የክፍያ ቀነ-ገደቦች
- **የዘገየ ክፍያ ቅጣት** — ዘግይቶ ለከፈለ የሚታከለው ተጨማሪ ክፍያ

### 9.5 የክፍያ በር (Payment Gateways)

ZENOVA ከሶስት ዋና ዋና የክፍያ መጋራቢያዎች ጋር ይዋሃዳል፦

**ቻፓ (Chapa)፦**
- የክሬዲት ካርድ ክፍያ
- የቴሌብር ክፍያ (በቻፓ በኩል)
- የአፕል ፔይ እና ሳምሰንግ ፔይ
- ዓለም አቀፍ የክፍያ መቀበያ

**ቴሌብር (Telebirr)፦**
- የሞባይል ገንዘብ ዝውውር
- የኢትዮቴሌኮም ደንበኞች ክፍያ
- ከቻፓ እና ከሲቢኢ ብር አማራጭ የክፍያ ዘዴ
- በቀጥታ ወደ ትምህርት ቤት የሞባይል ሒሳብ ክፍያ

**ሲቢኢ ብር (CBE Birr)፦**
- የንግድ ባንክ ደንበኞች የሞባይል ክፍያ
- የባንክ ሒሳብ ዝውውር
- ከሲቢኢ ቅርንጫፎች ክፍያ

### 9.6 የወጪ አስተዳደር (Expense Management)

የፋይናንስ ሞጁል የትምህርት ቤቱን ወጪ ለማስተዳደር የሚከተሉትን ያካትታል፦

**የወጪ ዓይነቶች፦**
- የሰራተኛ ደመወዝ እና ደሞዝ
- የፍጆታ ወጪ (መብራት፣ ውሃ፣ ኢንተርኔት)
- የጥገና ወጪ (ህንጻ፣ ተሽከርካሪ)
- የቢሮ ቁሳቁስ ግዢ
- የማስተማሪያ ቁሳቁስ ግዢ
- የሌሎች ወጪዎች

**የወጪ አስተዳደር ሂደት፦**
1. የወጪ ጥያቄ (Expense Request) መፍጠር
2. የወጪ ማረጋገጫ (Approval) ሂደት
3. የወጪ ምዝገባ እና ምደባ
4. የወጪ ሪፖርት እና ትንተና

### 9.7 የደመወዝ አስተዳደር (Payroll)

የደመወዝ አስተዳደር ክፍል የሚከተሉትን ያከናውናል፦

**የደመወዝ ውቅር፦**
- የእያንዳንዱ ሰራተኛ መሰረታዊ ደመወዝ
- የተለያዩ አበሎች (መጓጓዣ፣ መኖሪያ፣ ምግብ)
- የተከራይ ቀረጥ (የገቢ ግብር)
- የጡረታ መዋጮ
- የተለያዩ ቅነሳዎች

**የደመወዝ አከፋፈል፦**
- ወርሃዊ የደመወዝ ክፍያ ቀነ-ገደብ
- የደመወዝ ሉህ (Payslip) ማዘጋጀት
- የደመወዝ ሪፖርት ለሒሳብ መዛግብት
- የደመወዝ ታክስ ሪፖርት

### 9.8 የሂሳብ ሪፖርቶች (Financial Reports)

የፋይናንስ ሞጁል የሚከተሉትን ሪፖርቶች ያቀርባል፦

**ዕለታዊ ሪፖርቶች፦**
- የዕለቱ የገቢ ሪፖርት
- የዕለቱ የወጪ ሪፖርት
- የዕለቱ የጥሬ ገንዘብ ቀሪ ሂሳብ

**ወርሃዊ ሪፖርቶች፦**
- የወሩ አጠቃላይ የገቢ እና ወጪ ሪፖርት
- የወሩ የተማሪ ክፍያ ሪፖርት
- የወሩ የደመወዝ ሪፖርት
- የወሩ የቀሪ ሂሳብ ሪፖርት

**ዓመታዊ ሪፖርቶች፦**
- ዓመታዊ የሒሳብ በጀት እና ትክክለኛ አፈጻጸም
- ዓመታዊ የገቢ እና ወጪ ሒሳብ
- ዓመታዊ የሒሳብ ሚዛን ወረቀት
- የኦዲት ሪፖርት ዝግጅት

### 9.9 የዕዳ አስተዳደር (Debt Management)

የፋይናንስ ሞጁል የተማሪዎችን ዕዳ ለማስተዳደር ልዩ ባህሪ አለው፦

- የዕዳ ክትትል — ክፍያ ያልከፈሉ ተማሪዎችን መለየት
- የዕዳ ማስጠንቀቂያ — ለወላጆች አውቶማቲክ ማሳሰቢያ መላክ
- የዕዳ ክፍያ ዕቅድ — ከወላጆች ጋር የክፍያ ዕቅድ ማዘጋጀት
- የዕዳ ሪፖርት — ጠቅላላ የትምህርት ቤት ዕዳ ሪፖርት
- የዕዳ መሰረዝ — ሊሰበሰብ የማይችል ዕዳ መሰረዝ

### 9.10 የበጀት እቅድ (Budget Planning)

ትምህርት ቤቶች በጀታቸውን እንደሚከተለው ማቀድ ይችላሉ፦

- ዓመታዊ በጀት ማዘጋጀት
- በጀት በክፍሎች መከፋፈል (አካዳሚክ፣ አስተዳደር፣ ልማት)
- የበጀት አፈጻጸም ክትትል
- የበጀት ልዩነት ትንተና (ታቅዶ ከነበረው እና ከተከናወነው ጋር)
- የአዲስ ዓመት በጀት እቅድ

### 9.11 የፋይናንስ ባለሙያ ዳሽቦርድ

የፋይናንስ ባለሙያ ዳሽቦርድ የሚከተሉትን ያሳያል፦

- የዛሬው ጠቅላላ ገቢ እና ወጪ
- የተጠናቀቁ እና ያልተጠናቀቁ ክፍያዎች
- ወርሃዊ የገቢ ግራፍ
- ከፍተኛ ዕዳ ያለባቸው ተማሪዎች
- የቅርብ ጊዜ የክፍያ እንቅስቃሴ
- የደመወዝ ክፍያ ማሳሰቢያዎች

- የክፍል መርሐ ግብር መከታተል
- የክፍል ዝርዝር ማየት
- የተማሪ መገኘት ማስመዝገብ

**የፈተና አስተዳደር፦**
- ፈተና መፍጠር እና ማዘጋጀት
- የፈተና ውጤት ማስገባት
- የውጤት ስታቲስቲክስ ማየት
- የተማሪ ግምገማ ማድረግ

**የተማሪ ክትትል፦**
- የተማሪ መገኘት ሪፖርት ማየት
- የተማሪ ውጤት ታሪክ መከታተል
- ለወላጆች የተማሪ ሁኔታ ሪፖርት መላክ
- የተማሪ ባህሪ እና እንቅስቃሴ ማስመዝገብ

### 10.3 የመምህር ዳሽቦርድ

የመምህር ዳሽቦርድ የሚከተሉትን ያሳያል፦

- የዛሬው የትምህርት መርሐ ግብር
- ጠቅላላ የተማሪ ብዛት
- የዛሬው የተማሪ መገኘት መቶኛ
- ወደ ማስገባት የሚጠብቁ ውጤቶች
- የቅርብ ጊዜ ማሳሰቢያዎች
- የክፍል አፈጻጸም አጠቃላይ እይታ

### 10.4 የመገኘት ምዝገባ

መምህሩ የተማሪን መገኘት በተለያዩ መንገዶች ማስመዝገብ ይችላል፦

- በእጅ ማስመዝገብ — በሲስተሙ ገጽ ላይ እያንዳንዱን ተማሪ ምልክት ማድረግ
- በQR ኮድ — ተማሪዎች የራሳቸውን QR ኮድ በማሳየት መገኘታቸውን ማረጋገጥ
- በNFC ካርድ — ተማሪዎች የNFC ካርዳቸውን በማንበቢያ ላይ በማንጋት መገኘታቸውን ማረጋገጥ
- በቡድን ምዝገባ — የጠቅላላውን ክፍል መገኘት በአንድ ጊዜ ማስመዝገብ

- የውጤት መዛን (መቶኛ ክብደት) መወሰን

**የመገኘት ክትትል፦**

**የክፍያ አስተዳደር፦**

- በመስመር ላይ ክፍያ መክፈል (በቻፓ፣ ቴሌብር፣ ሲቢኢ ብር)
- የክፍያ ደረሰኝ ማውረድ

### 11.3 የወላጅ ዳሽቦርድ

የወላጅ ዳሽቦርድ የሚከተሉትን ያሳያል፦

- የሚከፈለው ቀሪ ሂሳብ
- ከትምህርት ቤት የተላኩ ማሳሰቢያዎች
- የትምህርት ቤት ክስተቶች እና ፕሮግራሞች

### 11.4 የወላጅ-ትምህርት ቤት ግንኙነት

ZENOVA ወላጆች ከትምህርት ቤቱ ጋር እንዲገናኙ የተለያዩ መንገዶችን ይሰጣል፦

- ከትምህርት ቤት አስተዳደር ጋር መገናኘት

**የአበዳሪ አስተዳደር፦**

- ያልተመለሱ መጻሕፍት ዝርዝር
- የአበዳሪ ማስጠንቀቂያ መላክ
- የተማሪ አበዳሪ ታሪክ

### 13.3 የቤተ-መጻሕፍት ዳሽቦርድ

የቤተ-መጻሕፍት ዳሽቦርድ የሚከተሉትን ያሳያል፦

- ደረሰኝ ማዘጋጀት
- የሽያጭ ሪፖርት

### 15.3 የመደብር ዳሽቦርድ

የመደብር ዳሽቦርድ የሚከተሉትን ያሳያል፦

- ጠቅላላ የእቃ ዓይነቶች
- ዝቅተኛ ክምችት ያላቸው እቃዎች
- የዛሬው ጠቅላላ ሽያጭ
- ከፍተኛ ሽያጭ ያላቸው እቃዎች
- የቅርብ ጊዜ ሽያጭ እንቅስቃሴ
- የክምችት ሁኔታ ሪፖርት

---

## ምዕራፍ ፲፮ — የNFC ሥርዓት (NFC System)

### 16.1 NFC ምንድን ነው?

NFC (Near Field Communication) በአጭር ርቀት የሚሰራ ሽቦ አልባ የመገናኛ ቴክኖሎጂ ነው። ZENOVA ውስጥ እያንዳንዱ ተማሪ የራሱ የሆነ የNFC ካርድ ይሰጠዋል ይህም የተማሪውን መለያ ቁጥር እና ሌሎች መረጃዎችን ይይዛል።

### 16.2 የNFC አጠቃቀም ቦታዎች

**የተማሪ መገኘት ምዝገባ፦**
- ተማሪው ወደ ትምህርት ቤት ሲገባ ካርዱን ያንኳኳል
- ከክፍል ሲወጣ ካርዱን ያንኳኳል
- ሲስተሙ አውቶማቲክ የመገኘት ምዝገባ ያደርጋል
- ወላጆች የልጃቸው የመግቢያ ሰዓት ማሳሰቢያ ያገኛሉ

**የካፍቴሪያ ክፍያ፦**
- ተማሪው ካርዱን በማንኳኳት ምግብ መግዛት ይችላል
- ክፍያው ከተማሪው የካፍቴሪያ ሒሳብ ይቀነሳል
- ወላጆች የልጃቸውን የካፍቴሪያ ወጪ መከታተል ይችላሉ

**የመደብር ግዢ፦**
- ተማሪው ካርዱን በመጠቀም ከመደብር መግዛት ይችላል
- ክፍያው ከተማሪው ሒሳብ ይቀነሳል

**የደህንነት ቁጥጥር፦**
- ተማሪው ወደ ትምህርት ቤት ሲገባ እና ሲወጣ ይመዘገባል
- ያልታወቀ ካርድ ከተጠቀመ ማንቂያ ይነሳል
- የተማሪ እንቅስቃሴ ሙሉ ምዝግብ ይኖረዋል

### 16.3 የNFC ካርድ አስተዳደር

**የካርድ ምዝገባ፦**
- ለእያንዳንዱ ተማሪ አዲስ ካርድ መስጠት
- የካርድ መረጃ ከተማሪ መረጃ ጋር ማያያዝ
- የካርድ መለያ ቁጥር (UID) ምዝገባ

**የካርድ አያያዝ፦**
- የጠፋ ካርድ ማገድ
- አዲስ ካርድ መተካት
- የካርድ ሁኔታ መከታተል (ንቁ/የታገደ/ጊዜው ያለፈ)
- የካርድ አጠቃቀም ታሪክ

### 16.4 የNFC ማንበቢያ መሣሪያዎች

ZENOVA ከሚከተሉት የNFC ማንበቢያ መሣሪያዎች ጋር ይሰራል፦

- በትምህርት ቤት መግቢያ ላይ የተገጠሙ ማንበቢያዎች
- በካፍቴሪያ ውስጥ የክፍያ ማንበቢያዎች
- በመደብር ውስጥ የክፍያ ማንበቢያዎች
- በቤተ-መጻሕፍት ውስጥ የአበዳሪ ማንበቢያዎች

---

## ምዕራፍ ፲፯ — የQR ኮድ ሥርዓት (QR Code System)

### 17.1 QR ኮድ ምንድን ነው?

QR (Quick Response) ኮድ በፍጥነት ሊነበብ የሚችል ባለ ሁለት ልኬት ባርኮድ ነው። ZENOVA ውስጥ እያንዳንዱ ተማሪ ከNFC ካርዱ በተጨማሪ የራሱ የሆነ ልዩ QR ኮድ አለው።

### 17.2 የQR ኮድ አጠቃቀም

- **የተማሪ መለያ** — በተማሪ መለያ ካርድ ላይ የታተመ QR ኮድ
- **የመገኘት ምዝገባ** — በQR ኮድ ስካነር መገኘትን ማስመዝገብ
- **የክፍያ ማረጋገጫ** — ከክፍያ ደረሰኝ ጋር QR ኮድ
- **የውሂብ መዳረሻ** — QR ኮዱን በማንበብ ወደ ተማሪው መረጃ መግባት

### 17.3 የQR ኮድ ትውልድ (Generation)

ሲስተሙ ለእያንዳንዱ ተማሪ ልዩ የሆነ QR ኮድ ያመነጫል። ይህ QR ኮድ የሚከተሉትን መረጃዎች ይይዛል፦

- የተማሪ መለያ ቁጥር (Student ID)
- የተማሪ ሙሉ ስም
- ክፍል እና ክፍልፋይ
- የተማሪ የትምህርት ቤት ኮድ

QR ኮዱ በሚከተሉት ቦታዎች ላይ ሊታተም ይችላል፦

- የተማሪ መለያ ካርድ
- የተማሪ ዩኒፎርም መለያ
- የውጤት ሪፖርቶች
- የክፍያ ደረሰኞች

---

## ምዕራፍ ፲፰ — የደህንነት ሥርዓት (Security System)

### 18.1 ሚና እና ሃላፊነት

የደህንነት ሞጁል የተማሪዎችን እና የሰራተኞችን ወደ ትምህርት ቤት መግባት እና መውጣት ይቆጣጠራል። የደህንነት ሰራተኞች የተማሪ እንቅስቃሴን በእውነተኛ ጊዜ መከታተል ይችላሉ።

### 18.2 ዋና ዋና ተግባራት

**የመግቢያ/መውጫ ቁጥጥር፦**
- ተማሪዎች ወደ ትምህርት ቤት ሲገቡ እና ሲወጡ መመዝገብ
- በNFC ካርድ ወይም QR ኮድ የተማሪ መለያ ማረጋገጥ
- ያልታወቁ ግለሰቦችን መለየት
- የጎብኝዎች ምዝገባ

**የክትትል ሥርዓት፦**
- በእውነተኛ ጊዜ የተማሪ እንቅስቃሴ መከታተል
- ያልተለመደ እንቅስቃሴ ማሳሰቢያ
- የተማሪ መገኛ ቦታ ክትትል (በNFC ነጥቦች)

**የሪፖርት አያያዝ፦**
- ዕለታዊ የመግቢያ/መውጫ ሪፖርት
- ያልተገኙ ተማሪዎች ዝርዝር
- የዘገዩ ተማሪዎች ሪፖርት
- የጎብኝዎች ምዝግብ

### 18.3 የደህንነት ዳሽቦርድ

የደህንነት ዳሽቦርድ የሚከተሉትን ያሳያል፦

- በአሁኑ ጊዜ በትምህርት ቤት ውስጥ ያሉ ተማሪዎች ብዛት
- የዛሬው የተማሪ መግቢያ እና መውጫ ስታቲስቲክስ
- የቅርብ ጊዜ የመግቢያ እንቅስቃሴ
- ያልተፈቀደ መዳረሻ ሙከራዎች
- የማንቂያ ማሳሰቢያዎች
- የወላጅ የልጅ መውጫ ፍቃድ ጥያቄዎች

---

## ምዕራፍ ፲፱ — የፍቃድ አሰጣጥ ሥርዓት (License System)

### 19.1 የፍቃድ ሥርዓት አጠቃላይ እይታ

የፍቃድ አሰጣጥ ሥርዓት ZENOVA እንዴት እንደሚሸጥ እና እንደሚሰራ የሚቆጣጠር ወሳኝ ክፍል ነው። እያንዳንዱ ትምህርት ቤት ሲስተሙን መጠቀም የሚችለው የሚሰራ ፍቃድ ካለው ብቻ ነው።

### 19.2 የፍቃድ ዓይነቶች

ZENOVA የተለያዩ የፍቃድ ደረጃዎችን ያቀርባል፦

**Basic፦**
- መሰረታዊ የተማሪ እና ሰራተኛ አስተዳደር
- የክፍያ አሰባሰብ
- የተማሪ መገኘት ክትትል
- ውሱን የተማሪ ብዛት

**Standard፦**
- ሁሉም የBasic ባህሪያት
- የቤተ-መጻሕፍት አስተዳደር
- የካፍቴሪያ ማገናኘት
- መሰረታዊ ሪፖርቶች
- NFC ድጋፍ

**Premium፦**
- ሁሉም የStandard ባህሪያት
- ሙሉ የፋይናንስ ሪፖርቶች
- የደመና ማመሳሰል
- የሞባይል መዳረሻ ለወላጆች
- የላቀ ደህንነት ባህሪያት
- የQR ኮድ ሥርዓት

### 19.3 የፍቃድ አሰራር ዘዴ

የፍቃድ አሰራሩ እንደሚከተለው ነው፦

1. **ፍቃድ መፍጠር** — ሱፐር አድሚኑ ለትምህርት ቤቱ አዲስ ፍቃድ ይፈጥራል
2. **ፍቃድ ማስረከብ** — የፍቃድ ቁልፍ (License Key) ለትምህርት ቤቱ ይሰጣል
3. **ፍቃድ ማስገባት** — ትምህርት ቤቱ ፍቃዱን በሲስተሙ ውስጥ ያስገባል
4. **ፍቃድ ማረጋገጥ** — ሲስተሙ ፍቃዱን ከማዕከላዊ ፍቃድ አገልጋይ ጋር ያረጋግጣል
5. **ፍቃድ መቆያ ጊዜ** — ፍቃዱ ለተወሰነ ጊዜ (ለምሳሌ 1 ዓመት) የሚሰራ ነው
6. **ፍቃድ ማደስ** — ጊዜው ሲያልፍ ፍቃዱ መታደስ አለበት

### 19.4 የፍቃድ ክትትል

ሲስተሙ የፍቃዶችን ሁኔታ እንደሚከተለው ይከታተላል፦

- ንቁ ፍቃዶች (Active)
- ጊዜያቸው ያለፈባቸው (Expired)
- ሊነቁ የሚችሉ (Pending Activation)
- የታገዱ (Suspended)
- ስለሚያልቅ ጊዜ አውቶማቲክ ማሳሰቢያ

## ምዕራፍ ፳ — የደመና ማመሳሰል (Cloud Sync)

### 20.1 የደመና ማመሳሰል ምንድን ነው?

የደመና ማመሳሰል የትምህርት ቤቱን የአካባቢ አገልጋይ ውሂብ ከZENOVA ደመና አገልጋይ ጋር በማዛመድ ውሂቡ በሁለቱም ቦታ እንዲገኝ የሚያደርግ ሂደት ነው።

### 20.2 የማመሳሰል ሂደት

የማመሳሰል ሂደቱ እንደሚከተለው ነው፦

- **አውቶማቲክ ማመሳሰል** — በየተወሰነ ጊዜ (ለምሳሌ በየ30 ደቂቃው) ሲስተሙ አውቶማቲክ ማመሳሰል ያከናውናል
- **የእጅ ማመሳሰል** — አስፈላጊ ሆኖ ሲገኝ የሲስተም አስተዳዳሪ በእጅ ማመሳሰል ይችላል
- **የመጀመሪያ ማመሳሰል** — አዲስ ትምህርት ቤት ሲመዘገብ ሙሉ የመጀመሪያ ማመሳሰል ያከናውናል
- **ከፊል ማመሳሰል** — የተለወጠውን ውሂብ ብቻ ማመሳሰል

### 20.3 የሚመሳሰሉ ውሂቦች

የሚከተሉት ውሂቦች በደመና ይመሳሰላሉ፦

- የተማሪ መረጃ (ስም፣ አድራሻ፣ ፎቶ፣ ወላጅ)
- የሰራተኛ መረጃ
- የክፍል እና ክፍልፋይ መዋቅር
- የክፍያ መረጃ
- የውጤት መረጃ
- የተማሪ መገኘት ሪፖርቶች
- የቤተ-መጻሕፍት አበዳሪ መረጃ
- የሲስተም ውቅሮች

### 20.4 የማመሳሰል ጥቅሞች

- **የውሂብ ምትኬ** — የትምህርት ቤቱ አገልጋይ ቢበላሽም ውሂቡ በደመና ይጠበቃል
- **ርቀት መዳረሻ** — ወላጆች እና ተማሪዎች ከቤት ውሂቡን ማየት ይችላሉ
- **ማዕከላዊ አስተዳደር** — ሱፐር አድሚን ሁሉንም ትምህርት ቤቶች ከአንድ ቦታ መቆጣጠር ይችላል
- **ውሂብ ትንተና** — የተለያዩ ትምህርት ቤቶችን አፈጻጸም ማነጻጸር

## ምዕራፍ ፳፩ — የሲስተም አሰማራት (Deployment)

### 21.1 የሲስተም አሰማራት አጠቃላይ እይታ

ZENOVA ሲስተም በሶስት ዋና ዋና ክፍሎች ይሰማራል፦ የደመና አገልጋይ (Cloud VPS)፣ የትምህርት ቤት አገልጋይ (School Ubuntu Server) እና የተጠቃሚ ፊት ለፊት (Web Frontend)።

### 21.2 የደመና አገልጋይ አሰማራት

የደመና አገልጋይ በሚከተሉት ደረጃዎች ይዘጋጃል፦

1. **VPS ውቅር** — ምናባዊ የግል አገልጋይ መግዛት እና ማዋቀር
2. **ዳታቤዝ ውቅር** — ማዕከላዊ የውሂብ ጎታ ማዋቀር
3. **Backend አሰማራት** — የኋላ-ተናጋሪ አገልግሎቶችን መጫን እና ማዋቀር
4. **Frontend አሰማራት** — የተጠቃሚ በይነገጽ ማሰማራት
5. **ደህንነት ውቅር** — ፋየርዎል፣ SSL ሰርተፍኬት እና ሌሎች የደህንነት ውቅሮች

### 21.3 የትምህርት ቤት አገልጋይ አሰማራት

እያንዳንዱ ትምህርት ቤት በራሱ ተቋም ውስጥ የሚከተሉትን ያዋቅራል፦

1. **Ubuntu ማዋቀር** — የኡቡንቱ ስርዓተ ክወና መጫን እና ማዋቀር
2. **Docker ውቅር** — አፕሊኬሽኑን በDocker ኮንቴይነሮች ማስኬድ
3. **የአካባቢ ዳታቤዝ** — የትምህርት ቤቱ የአካባቢ ዳታቤዝ ማዋቀር
4. **NFC/QR ማንበቢያ ውቅር** — የሃርድዌር መሣሪያዎችን ማዋቀር
5. **ኔትወርክ ውቅር** — ከደመና ጋር የሚገናኝ ኔትወርክ ማዋቀር

### 21.4 የሲስተም ጥገና

ሲስተሙን ለማስኬድ የሚከተሉት የጥገና ስራዎች ያስፈልጋሉ፦

- **ወቅታዊ ምትኬ (Backup)** — የዳታቤዝ እና የፋይሎች ምትኬ
- **የሶፍትዌር ማዘመን** — የሲስተሙን እና የሌሎች ሶፍትዌሮች ዝማኔ
- **ደህንነት ክትትል** — የሲስተሙን ደህንነት መከታተል
- **የአፈጻጸም ክትትል** — የአገልጋይ አፈጻጸም መከታተል
- **የስህተት አያያዝ** — ስህተቶችን መለየት እና ማስተካከል

## ምዕራፍ ፳፪ — ዕለታዊ የእውነተኛ ሕይወት ምሳሌ (Daily Real-World Example)

### 22.1 የተሟላ የአንድ ቀን ሂደት

የሚከተለው በZENOVA ሲስተም የሚተዳደር የአንድ ትምህርት ቤት የተለመደ ቀን እንዴት እንደሚሄድ ያሳያል፦

**ጠዋት 7፡30 — ተማሪዎች ወደ ትምህርት ቤት ይገባሉ**
- አበበ (የ12ኛ ክፍል ተማሪ) ወደ ትምህርት ቤት ደረሰ
- የNFC ካርዱን በመግቢያ ላይ ባለው ማንበቢያ ላይ አንኳኳ
- ሲስተሙ መግባቱን መዘገበ
- የደህንነት ሰራተኛው በዳሽቦርዱ ላይ አበበ እንደገባ አየ
- ለአበበ ወላጆች "ልጅዎ በሰዓቱ ትምህርት ቤት ገብቷል" የሚል SMS ደረሳቸው

**ጠዋት 8፡00 — የመጀመሪያ ትምህርት**
- ወ/ሮ አስቴር (የአበበ አስተማሪ) የክፍል ዝርዝሩን በሲስተሙ ከፈተች
- የቀኑን መገኘት ለመመዝገብ ጀመረች
- አበበ ጨምሮ ሁሉም ተማሪዎች እንደተገኙ ምልክት አደረገች
- አንድ ተማሪ (አለም) ያልተገኘ በመሆኑ ሲስተሙ ለወላጆቹ ማሳሰቢያ ላከ

**ቀኑ 10፡30 — የእረፍት ሰዓት**
- አበበ ወደ ካፍቴሪያ ሄደ
- የNFC ካርዱን በካፍቴሪያ ማንበቢያ አንኳኳ
- ሲስተሙ የካፍቴሪያ ሒሳቡን አረጋገጠ (በቂ ገንዘብ አለው)
- አበበ ምግብ መረጠ — ሲስተሙ 50 ብር ከሒሳቡ ቀነሰ
- የካፍቴሪያ ሰራተኛው ሽያጩን በዳሽቦርዱ ላይ አየ

**ቀኑ 12፡00 — የፋይናንስ ክፍል**
- የተማሪ ክፍያ ወቅት ነው
- አንድ ወላጅ የልጁን ክፍያ በቴሌብር ከፈለ
- የፋይናንስ ሰራተኛው ክፍያው እንደገባ በሲስተሙ አየ
- የክፍያ ደረሰኝ አውቶማቲክ ተዘጋጅቶ ለወላጁ ተላከ
- የተማሪው የክፍያ ሁኔታ ከ "ያልተከፈለ" ወደ "ተከፍሏል" ተለወጠ

**ከሰዓት 2፡00 — የውጤት ልቀቅ**
- ወ/ሮ አስቴር የቅርብ ጊዜ ፈተና ውጤቶችን አስገባች
- አበበ 85% አግኝቷል
- ሲስተሙ የአበበን አጠቃላይ አፈጻጸም አስልቶ አዘመነ
- ወላጆቹ የልጃቸውን ውጤት በሞባይላቸው አዩ

**ከሰዓት 4፡30 — የትምህርት ቤት መውጫ**
- አበበ ወደ ቤት ሲሄድ ካርዱን በመውጫ ማንበቢያ አንኳኳ
- ሲስተሙ መውጣቱን መዘገበ
- ለወላጆቹ "ልጅዎ በሰዓቱ ከትምህርት ቤት ወጥቷል" የሚል ማሳሰቢያ ደረሳቸው

**ማታ 8፡00 — የደመና ማመሳሰል**
- የትምህርት ቤቱ ውሂብ ከደመና ጋር ተሰማማ
- የዛሬው መገኘት፣ የካፍቴሪያ ሽያጭ፣ የክፍያ እና የውጤት ውሂብ ሁሉ ወደ ደመና ተላከ
- የሱፐር አድሚን ዳሽቦርድ በራሱ ታደሰ

### 22.2 የተለያዩ ባለድርሻ አካላት የቀኑ መጨረሻ ሪፖርቶች

- ያልተገኙ ተማሪዎች ዝርዝር
- አዲስ የተመዘገቡ ተማሪዎች

- ለወላጆች የተሰራ ሞባይል አፕ — የልጅን መረጃ በቀላሉ መከታተል
- ለመምህራን ሞባይል አፕ — መገኘት እና ውጤት ከስልክ ማስገባት
- ለተማሪዎች ሞባይል አፕ — የራስን መረጃ ማየት

**የላቀ ሪፖርት እና ትንተና (Advanced Analytics)፦**
- AI የተማሪ አፈጻጸም ትንበያ
- የራስ-ሰር ሪፖርት ማመንጨት
- የውሂብ ቪዥዋላይዜሽን ዳሽቦርዶች
- የተማሪ ፍላጎት እና ድክመት ትንተና

**የመስመር ላይ ትምህርት (Online Learning)፦**
- የቪዲዮ ትምህርት ክፍል
- የመስመር ላይ ፈተና አሰጣጥ
- የተማሪ-መምህር የመስመር ላይ ግንኙነት
- የትምህርት ቁሳቁስ መጋሪያ

**የላቀ ደህንነት ባህሪያት፦**
- የፊት መለያ ቴክኖሎጂ (Face Recognition)
- የአልትራሳውንድ መከታተያ
- የባዮሜትሪክ ማረጋገጫ
- 2-ደረጃ ማረጋገጫ (2FA)

**ሌሎች ባህሪያት፦**
- የወላጅ-አስተማር ስብሰባ አስተዳደር
- የትምህርት ቤት ክስተቶች እና ፕሮግራሞች አስተዳደር
- የስፖርት እና ከመደበኛ ትምህርት ውጪ እንቅስቃሴዎች
- የተማሪ ምክር እና የስነ-ልቦና ክትትል
- የህትመት ማዕከል (ተማሪ መለያ ካርድ፣ የውጤት ካርድ፣ የምስክር ወረቀት)

### 23.2 የአርክቴክቸር ክፍተቶች እና የመፍትሄ ምክሮች

የሚከተሉት በአሁኑ የሲስተም አርክቴክቸር ውስጥ ያሉ ተግዳሮቶች እና ክፍተቶች ናቸው። እነዚህ ከአሁኑ የኮድ ትንተና እና ከፕሮጀክቱ ሁኔታ አንጻር የተመደቡ ናቸው።

#### ከፍተኛ ቅድሚያ (High Priority)

**1. የUnit እና Integration ሙከራ (Test) እጥረት**
- **ችግሩ:** ሲስተሙ 229 ገፆች ቢኖረውም አጠቃላይ የሙከራ ሽፋን (test coverage) በጣም ዝቅተኛ ነው። ይህ አዳዲስ ባህሪያት ሲጨመሩ ወይም ነባር ባህሪያት ሲቀየሩ ስህተቶችን ለመለየት ያስቸግራል።
- **ምክረ ሀሳብ:** የJest እና React Testing Library ሙከራዎች ለዋና ዋና ተግባራት መፃፍ አለባቸው።

**2. የስህተት አያያዝ (Error Handling) አለመሟላት**
- **ችግሩ:** በአብዛኛዎቹ ገፆች ላይ የአገልጋይ ስህተቶችን ለተጠቃሚ ለማሳወቅ ወጥ የሆነ ዘዴ የለም። አብዛኛው ጊዜ ስህተቶች ለተጠቃሚ ሳይታወቅ ይቀራሉ።
- **ምክረ ሀሳብ:** ወጥ የሆነ የስህተት አያያዝ ስልት (Error Boundary፣ Toast Notifications) መዘርጋት አለበት።

**3. የውሂብ ማረጋገጫ (Input Validation) አለመኖር**
- **ችግሩ:** በብዙ ቅጾች ላይ የግብዓት ማረጋገጫ በደንብ አልተሰራም። ይህ የተሳሳተ ውሂብ ወደ ዳታቤዝ እንዲገባ ያደርጋል።
- **ምክረ ሀሳብ:** የZod ወይም Yup በመጠቀም የቅጽ ማረጋገጫ መዘርጋት አለበት።

#### መካከለኛ ቅድሚያ (Medium Priority)

**4. የLoading እና Skeleton ሁኔታዎች አለመኖር**
- **ችግሩ:** ውሂብ በሚጫንበት ጊዜ ለተጠቃሚ ምንም ዓይነት የጭነት አመልካች (loading spinner/skeleton) አይታይም።
- **ምክረ ሀሳብ:** React Query እየተጠቀምን ስለሆነ የisLoading እና isFetching ሁኔታዎችን በመጠቀም የSkeleton ክፍሎች መዘርጋት አለባቸው።

**5. የተከፋፈለ የውሂብ አመንጪ (State Management)**
- **ችግሩ:** አንዳንድ ጊዜ የአካባቢ ሁኔታ (local state) እና የአገልጋይ ሁኔታ (server state) ይደባለቃሉ።
- **ምክረ ሀሳብ:** የReact Query ካሽ ማዘመን እና የlocal state አጠቃቀም ወጥ ደንብ ማውጣት።

**6. የሞባይል ምላሽ ሰጪነት (Responsive Design) ችግሮች**
- **ችግሩ:** አንዳንድ ገፆች በሞባይል መሣሪያዎች ላይ በደንብ አይታዩም።
- **ምክረ ሀሳብ:** የተረፈውን የፊት-ለፊት ክፍል በሙሉ በTailwind responsive classes ማሻሻል።

#### ዝቅተኛ ቅድሚያ (Low Priority)

**7. የAPI ዘገባ አለመኖር (API Documentation)**
- **ችግሩ:** የAPI ኤንድፖይንቶች ሰነድ የላቸውም።
- **ምክረ ሀሳብ:** Swagger/OpenAPI ዘገባ ማዘጋጀት።

**8. የአፈጻጸም መለኪያ (Performance Metrics)**
- **ችግሩ:** የገፅ ጭነት ጊዜ እና የAPI ምላሽ ጊዜ የሚለካበት ዘዴ የለም።
- **ምክረ ሀሳብ:** Lighthouse ወይም Web Vitals መጠቀም።

**9. የEnd-to-End ሙከራ (E2E Testing) እጥረት**
- **ችግሩ:** የCypress ወይም Playwright ሙከራዎች የሉም።
- **ምክረ ሀሳብ:** ለዋና ዋና የተጠቃሚ ጉዞዎች E2E ሙከራ መፃፍ።

**10. የTypeScript ማሻሻያ**
- **ችግሩ:** 776 የ`no-explicit-any` ማስጠንቀቂያዎች አሁንም አሉ።
- **ምክረ ሀሳብ:** ወሳኝ በሆኑ ቦታዎች ላይ እነዚህን ማስጠንቀቂያዎች በማስተካከል የTypeScript ደህንነት ማሻሻል።

ZENOVA የትምህርት ተቋማትን ሁሉንም የአስተዳደር ስራዎች በአንድ ላይ የሚያስተናግድ ሙሉ የተቀናጀ የትምህርት ቴክኖሎጂ መድረክ ነው። በሶስት-ደረጃ አርክቴክቸር፣ ለ13 ሚናዎች የተለዩ ዳሽቦርዶች፣ የNFC/QR ቴክኖሎጂ ውህደት እና ከበርካታ የክፍያ መጋራቢያዎች ጋር የተዋሃደ ሲሆን ከ229 በላይ ገፆችን ያቀፈ ነው።
